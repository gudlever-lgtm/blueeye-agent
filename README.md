# BlueEye Agent

A lightweight network probe that runs **on (or beside) a Cisco switch** and
feeds connectivity data to the [BlueEye server](https://github.com/gudlever-lgtm/blueeye-server).
It registers itself, polls the server for jobs, runs network checks, and
reports the results. Operators use the server dashboard to troubleshoot
outages on specific locations and connections.

## How it works

1. On startup the agent reads its identity (Common Name) from its client
   certificate and registers with the server over mutual TLS.
2. Every `POLL_INTERVAL` seconds it asks the server for pending jobs.
3. Each job runs a built-in check; the result (or error) is posted back.
4. A failing check never crashes the agent — the error is logged, reported,
   and polling continues.

## Built-in checks

| Type   | Purpose                          | Useful `params`                       |
|--------|----------------------------------|---------------------------------------|
| `ping` | ICMP reachability, latency, loss | `count`                               |
| `dns`  | DNS resolution + timing          | `record`, `nameserver`                |
| `http` | HTTP(S) endpoint health          | `method`, `timeout`, `expect_status`  |

## Configuration

Set via environment variables (see `.env.example`):

| Variable             | Default                   | Description                          |
|----------------------|---------------------------|--------------------------------------|
| `BLUEEYE_SERVER_URL` | `https://localhost:8443`  | Server mTLS endpoint                 |
| `CLIENT_CERT`        | `certs/agent-001.crt`     | Client certificate (CN = identity)   |
| `CLIENT_KEY`         | `certs/agent-001.key`     | Client private key                   |
| `CA_CERT`            | `certs/ca.crt`            | CA used to verify the server         |
| `POLL_INTERVAL`      | `30`                      | Seconds between job polls            |
| `AGENT_LOCATION`     | `unknown`                 | Physical location (shown on dashboard)|
| `AGENT_CONNECTION`   | `unknown`                 | Uplink / connection name             |

## Run locally

```bash
pip install -r requirements.txt
cp .env.example .env          # edit paths and location
set -a; . ./.env; set +a
python main.py
```

Certificates are generated by the `scripts/gen_certs.sh` script in the
**blueeye-server** repository.

## Run with Docker

The full stack (server + database + agent) is orchestrated from the
**blueeye-server** repository's `docker-compose.yml`. Clone both repositories
as siblings so Compose can build the agent image:

```
parent/
├── blueeye-server/
└── blueeye-agent/
```

Then from `blueeye-server/`: `docker compose up --build`.

## Install on a Cisco switch

Cisco IOS-XE devices with a Guest Shell (or any attached Linux host) can run
the agent as a systemd service:

```bash
sudo mkdir -p /opt/blueeye-agent
sudo cp -r agent main.py requirements.txt /opt/blueeye-agent/
sudo cp .env.example /opt/blueeye-agent/.env   # then edit it
sudo pip3 install -r /opt/blueeye-agent/requirements.txt
sudo useradd -r -s /usr/sbin/nologin blueeye
sudo cp install/blueeye-agent.service /etc/systemd/system/
sudo systemctl enable --now blueeye-agent
```

Place the agent's certificate, key, and the CA certificate where `.env`
points (`/opt/blueeye-agent/certs/` by default).
